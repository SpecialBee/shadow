1. NPC 종류
================================================================

  NpcType.Guard    — FSM 기반 경비병. 시야 안에 들어오면 내부 의심도 게이지 상승.
  NpcType.Civilian — 전역 의심도(SuspicionManager.CurrentSuspicion)에 반응하는 일반 NPC.

  NPCController 컴포넌트 > "NPC 종류" 헤더에서 선택.

================================================================
2. Guard FSM (상태 머신)
================================================================

  상태 전이 흐름:
    Idle ──→ Patrol ──→ Suspicious ──→ Alert ──→ Chase ──→ Search ──→ Patrol/Idle
                ↑___________________________________|

  각 상태 설명:
    Idle        : 순찰 포인트 없을 때 정지 대기.
    Patrol      : 순찰 포인트 순환 이동. 플레이어 발견 시 GainSuspicion 호출.
    Suspicious  : 플레이어 주시, 내부 의심도(_suspicion) 누적. alertThreshold 초과 시 Alert 진입.
    Alert       : 이동하며 추적. 내부 의심도 빠르게 누적. chaseThreshold 초과 시 Chase 진입.
    Chase       : 플레이어 직접 추격. 시야 잃으면 sightLoseDelay 후 Search 진입.
    Search      : 마지막 위치 수색. searchDuration 후 Patrol/Idle 복귀.

  말풍선:
    Suspicious (직접 발각) → "음...?"
    Suspicious (전파 수신) → "어디야?!"   ← 경보 전파를 받은 경우 별도 표시
    Alert      → "거기 누구야?!"
    Chase      → "잡아라!"
    Search     → "어디 갔지..."
    복귀 시    → "...착각이었나"

  의심도 초기화 동작:
    Search → Patrol/Idle 복귀 시 _suspicion = alertThreshold × 0.3 (완전 리셋 아님).
    이전 조우의 기억을 약하게 유지 → 재발각 시 Alert까지 약 0.93초 (기본 1.33초 대비 단축).

  PlayerExposureTracker 연동:
    Suspicious 진입 → RegisterSoftThreat → ExposureState.ExposedClose (+5/s)
    Alert/Chase 진입 → RegisterNpcThreat → ExposureState.ExposedSight (+20/s)
    상태 이탈 시 자동 해제

  초기 시선 방향:
    Start()에서 첫 번째 순찰 포인트 방향으로 자동 설정.
    순찰 포인트 없으면 Vector2.right 기본값.

3. Civilian 의심도 구간 시스템
================================================================

  전역 의심도(0~100)를 4구간으로 나눠 반응:

    구간 1 (의심도 0 ~ tier2At):        조용히 감시 — 머리 위 "◉" 아이콘 표시
    구간 2 (의심도 tier2At ~ tier3At):  "?" 말풍선 표시
    구간 3 (의심도 tier3At ~ tier4At):  "뭐지?" + 추격 시작
    구간 4 (의심도 tier4At ~ 100):      "그림자가 없다!" + 빠른 추격

  기본 경계값 (Inspector에서 NPC별 조정 가능):
    tier2At = 25   tier3At = 50   tier4At = 75

  플레이어를 시야 안에서 볼 때마다:
    → AddSuspicion(civilianSightRate × 복합배율 × Time.deltaTime)
    → 구간 변화 시 자동으로 반응 전환

  시야를 잃으면 구간 0(리셋) → 순찰 복귀
  Chase 중 시야 잃고 sightLoseDelay 초 지나면 순찰 복귀

  경계 레벨 기여:
    tier 3 또는 4 진입 시 AlertManager.RegisterCivilianAlert() → +1 포인트.
    단, 같은 Civilian이 30초 이내 중복 기여 불가 (쿨다운).

4. 고도화 시스템
================================================================

  ── A. 정보 전파 (Alert Spreading) ──────────────────────────

  Guard가 Alert/Chase 진입 시:
    → 반경 alertBroadcastRadius(기본 5m) 내 다른 Guard에게 브로드캐스트
    → 수신한 Guard: lastKnown 위치 공유받고 즉시 Suspicious 상태 진입 + "어디야?!" 표시
    → 이미 Alert/Chase 중인 Guard는 수신 무시

  Civilian이 구간 3 or 4 진입 시 (Chase 시작할 때):
    → 반경 alertBroadcastRadius 내 다른 Civilian에게 브로드캐스트
    → 수신한 Civilian: 플레이어를 보고 있을 경우 현재 구간 +1 즉시 상승

  Inspector 설정: Alert Broadcast Radius (각 NPCController마다 설정)

  ── B. 복합 승수 (Crowd Multiplier) ─────────────────────────

  동시에 플레이어를 보는 Civilian 수에 따라 의심도 상승률 증가:
    1명 → ×1.0 (기본)
    2명 → ×1.3
    3명 → ×1.6
    4명 이상 → ×2.0 (상한)

  PlayerExposureTracker.GetCrowdMultiplier()로 계산.
  Civilian이 시야 ON/OFF 시 자동으로 카운터 등록/해제.

  AlertLevel 4 배율과 스택됨 (예: 레벨4 + 2명 = ×2.0 × 1.3 = ×2.6)

  피드백: 동시 감시 인원이 2명 이상이면 의심도 바가 맥동함 (인원 많을수록 빠름).

  ── D. 전역 경계 레벨은 섹션 5 참조 ─────────────────────────

5. 전역 경계 레벨 (AlertManager)
================================================================

  AlertManager 싱글턴이 포인트를 누적해 레벨 결정.
  레벨은 한 번 올라가면 내려가지 않음 (세션 내 지속 긴장감).

  포인트 획득:
    Guard가 Alert/Chase 진입할 때마다:  +2 포인트
    Civilian이 tier 3/4 진입할 때마다: +1 포인트 (30초 쿨다운, NPC당)
    의심도 70 이상 상태 10초 지속마다: +1 포인트

  레벨 임계:
    레벨 1: 0~1 포인트
    레벨 2: 2~3 포인트
    레벨 3: 4~6 포인트
    레벨 4: 7+ 포인트

  레벨별 효과:
    레벨 1 — 기본 상태. 변화 없음.

    레벨 2 — 초기 경계
      · 전 NPC 이동 속도 ×1.3
      · Civilian 시야 범위 ×1.2

    레벨 3 — 적극 수색
      · 전 NPC 이동 속도 ×1.3 (레벨 2와 동일)
      · Guard sightLoseDelay ×0.5 (시야 잃어도 오래 추격)
      · Guard searchDuration ×2.0 (수색 시간 두 배)
      · Civilian 구간 경계값 -15 (더 낮은 의심도에서 반응)

    레벨 4 — 최고 경계
      · 전 NPC 이동 속도 ×1.6
      · 의심도 자연 감소율 80% 차단 (그림자 안에 있어도 거의 안 줄어듦)
      · Civilian 의심도 상승률 ×2.0 (복합 승수와 스택)

  피드백 (→ 섹션 7 참조):
    · HUD 우측 상단 아이콘 색상으로 레벨 상시 표시
    · 레벨 상승 시 화면 중앙 알림 텍스트 + 플래시
    · 레벨 3/4: 화면 주변 비네팅 적용

  씬 세팅: SuspicionManager 오브젝트에 AlertManager 컴포넌트 부착 (완료).

6. 체포 판정
================================================================

  Chase 상태의 NPC가 플레이어 오브젝트에 물리 충돌 시 즉시 체포.
    OnCollisionEnter2D (일반 콜라이더 충돌)
    OnTriggerEnter2D   (트리거 콜라이더 충돌)
    → SuspicionManager.TriggerArrest() → GameOverReason.Arrested

  의심도 100 도달 시 게임오버:
    → GameOverReason.SuspicionFull

  게임오버 화면에서 원인 텍스트 표시:
    체포됨     → "NPC에게 발각되었습니다"
    의심도 MAX → "의심도가 최대에 달했습니다"

7. 플레이어 피드백 시스템
================================================================

  ── 경계 레벨 HUD (AlertLevelUI.cs) ─────────────────────────

  · HUD 우측 상단: 아이콘 + 숫자로 현재 레벨 상시 표시.
    색상: 레벨1=흰 / 레벨2=노랑 / 레벨3=주황 / 레벨4=빨강.
  · 레벨 상승 시 화면 중앙에 "경계 레벨 N" 알림 텍스트.
    2.0초 유지 후 0.5초에 걸쳐 페이드아웃.
  · 레벨 상승 순간 화면 플래시 (레벨 색상, 0.35초).
  · AlertManager.OnAlertLevelChanged 이벤트 구독.

  ── Civilian 감시 아이콘 (NPCController.cs) ──────────────────

  · Civilian이 tier 1 구간에서 플레이어를 볼 때 머리 위 "◉" 표시.
  · 말풍선이 없는 조용한 감시 구간에서 플레이어가 알아챌 수 있는 단서.
  · tier 2 이상 진입 시 아이콘 숨김 (말풍선이 대체).
  · 시야를 잃으면 즉시 숨김.

  ── 의심도 바 맥동 (SuspicionUI.cs) ─────────────────────────

  · 동시에 플레이어를 감시 중인 Civilian이 2명 이상이면 fill bar 깜빡임.
  · 깜빡임 주기: 2~5 Hz (감시 인원이 많을수록 빠름).
  · 복합 승수(B 시스템)가 활성화되어 있음을 시각적으로 표현.

  ── Guard 전파 수신 말풍선 (NPCController.cs) ────────────────

  · BroadcastAlert를 받고 Suspicious 진입한 Guard는 "어디야?!" 표시.
  · 직접 발각 시 "음...?"과 구분되어 플레이어가 전파 상황을 인지 가능.

  ── 화면 비네팅 (VignetteController.cs) ─────────────────────

  · AlertManager 레벨에 따라 URP Volume의 Vignette intensity 조절.
    레벨 1/2: 0 (없음)
    레벨 3:   0.28 (약하게)
    레벨 4:   0.48 (강하게)
  · 부드러운 보간으로 전환 (lerpSpeed = 2).
  · 씬에 GlobalPostProcess 오브젝트에 Volume + VignetteController 부착.

8. Inspector 설정 항목 (NPCController)
================================================================

  [기본 설정]
    Data              : NpcKindData ScriptableObject 연결 (필수)
    Patrol Points     : 순찰 경로 Transform 배열 (첫 포인트가 초기 시선 방향 결정)
    Obstacle Layer    : 벽 레이어 (벽 회피용)

  [NPC 종류]
    Npc Type          : Guard / Civilian 선택

  [일반 NPC — 의심도 구간 경계] (Civilian 전용)
    Civilian Tier2 At  : 기본 25  (? 말풍선 시작 의심도)
    Civilian Tier3 At  : 기본 50  (추격 시작 의심도 / 경계 레벨 +1pt)
    Civilian Tier4 At  : 기본 75  (빠른 추격 시작 의심도 / 경계 레벨 +1pt)
    Civilian Fast Mult : 기본 1.6 (4구간 추격 속도 배율)
    Civilian Sight Rate: 기본 3   (시야 내 의심도 증가량/초)

  [정보 전파]
    Alert Broadcast Radius: 기본 5 (브로드캐스트 반경 m)

  [NpcKindData ScriptableObject]
    viewAngle           : 시야각 (도)
    viewRange           : 시야 거리
    suspicionGainRate   : Guard 내부 의심도 상승속도 (기본 30/s)
    suspicionDecayRate  : Guard 내부 의심도 감소속도 (기본 8/s)
    alertThreshold      : Suspicious→Alert 임계값 (기본 40)
    chaseThreshold      : Alert→Chase 임계값 (기본 75)
    patrolSpeed         : 순찰 속도 (기본 2)
    chaseSpeed          : 추격 속도 (기본 4)
    sightLoseDelay      : 시야 잃은 후 포기까지 대기 시간 (기본 2s)
    searchDuration      : Search 상태 지속 시간 (기본 5s)
    closeRange          : 즉시 Chase 전환 근접 거리 (기본 2m)
    arrestTime          : (미사용 — 충돌 기반 체포로 전환됨)

  [AlertLevelUI 레퍼런스]
    Level Icon          : HUD 아이콘 Image
    Level Number        : 레벨 숫자 TextMeshProUGUI
    Notification Text   : 알림 텍스트 TextMeshProUGUI
    Flash Overlay       : 전체 화면 플래시 Image

9. 씬 세팅 체크리스트
================================================================

  필수 컴포넌트:
    □ GameLoopController   — 씬에 하나 존재
    □ SuspicionManager     — 씬에 하나 존재
    □ AlertManager         — SuspicionManager 오브젝트에 부착 (완료)
    □ PlayerExposureTracker — Player 오브젝트에 부착
    □ GameOverUI           — UICanvas에 부착

  UI 컴포넌트:
    □ AlertLevelWidget     — HUDPanel 하위, AlertLevelUI 컴포넌트 부착 (완료)
    □ AlertNotification    — Canvas 직속 자식, TextMeshProUGUI (완료)
    □ AlertFlash           — Canvas 직속 자식, Image 전체 화면 (완료)
    □ GlobalPostProcess    — 씬 루트, Volume + VignetteController 부착 (완료)

  Guard NPC 세팅:
    □ NPCController > NpcType = Guard
    □ NpcKindData 에셋 연결 (5.ScriptableObjects/NpcKindData_Guard.asset)
    □ Obstacle Layer 설정
    □ Patrol Points 연결 (첫 포인트 = 초기 시선 방향)
    □ Rigidbody2D > Body Type = Kinematic
    □ Collider2D 부착 (충돌 체포용)
    □ SpeechBubble 컴포넌트 > Font = DungGeunMo SDF

  Civilian NPC 세팅:
    □ NPCController > NpcType = Civilian
    □ NpcKindData 에셋 연결 (2.Asset/Player&NPC/NpcKindData_citizen.asset)
    □ 구간 경계값 조정 (NPC별 예민도 차별화 가능)
    □ Rigidbody2D > Body Type = Kinematic
    □ Collider2D 부착 (충돌 체포용)
    □ SpeechBubble 컴포넌트 > Font = DungGeunMo SDF

  레이어 설정:
    □ Player 태그 = "Player"
    □ 벽 오브젝트 레이어 → obstacleLayer에 지정
    □ ShadowZone 레이어 → PlayerExposureTracker > shadowLayer에 지정

  에셋 현황 (활성):
    Guard 데이터  : Assets/5.ScriptableObjects/NpcKindData_Guard.asset
    Civilian 데이터: Assets/2.Asset/Player&NPC/NpcKindData_citizen.asset
    폰트          : Assets/3.Fonts/DungGeunMo SDF.asset
