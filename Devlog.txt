  2026-06-11  (세션 1) — NPC 시스템 설계 및 구현
================================================================

  ── 빛/그림자 의심도 연동 수정 ──────────────────────────────
  · 플레이어가 빛 근처에 있어도 의심도가 오르지 않던 버그 수정.
    원인: PlayerExposureTracker.Evaluate()에 Lit 분기 누락.
    수정: _litCount > 0 조건 추가, rateLit = 8f SerializeField 공개.

  ── NPC 이중 구조 구현 ──────────────────────────────────────
  · NpcType 열거형 추가 (Guard / Civilian).
  · Guard: 기존 FSM 유지 (Idle→Patrol→Suspicious→Alert→Chase→Search).
  · Civilian: 전역 의심도 4구간 반응 시스템 신규 구현.
      구간 1 — 조용히 감시 (말풍선 없음)
      구간 2 — "?" 말풍선
      구간 3 — "뭐지?" + 추격
      구간 4 — "그림자가 없다!" + 빠른 추격
  · Civilian 시야 내 의심도 상승률(civilianSightRate) Inspector 공개.
  · Civilian 추격 중에도 AddSuspicion 유지 (이전 버그: 50에서 멈춤).

  ── 충돌 기반 체포 시스템 ───────────────────────────────────
  · 기존 타이머 기반 체포 → OnCollisionEnter2D / OnTriggerEnter2D 방식으로 변경.
  · Chase 상태 NPC가 Player 태그 오브젝트와 충돌 시 즉시 TriggerArrest().
  · GameOverReason 열거형 추가 (SuspicionFull / Arrested).
  · GameOverUI에 원인 텍스트 표시 ("NPC에게 발각되었습니다" / "의심도가 최대에 달했습니다").

  ── 고도화 시스템 A/B/D 구현 ────────────────────────────────
  A. 정보 전파 (Alert Spreading)
     Guard Alert/Chase 진입 → 반경 내 Guard에 lastKnown 공유 + Suspicious 진입.
     Civilian tier 3/4 진입 → 반경 내 Civilian의 tier +1 상승 (시야 중일 때만).

  B. 복합 승수 (Crowd Multiplier)
     동시 감시 Civilian 수에 따라 의심도 상승률 배율 (×1.0 ~ ×2.0).
     PlayerExposureTracker에서 카운트 관리.

  D. 전역 경계 레벨 (AlertManager)
     AlertManager 싱글턴 신규 생성.
     Guard Alert 진입 +2pt / 고의심도(70+) 10초 지속 +1pt.
     레벨 1~4, 한 번 오르면 내려가지 않음.
     레벨별 NPC 속도/시야/감소율 변화 적용.

  ── 기타 버그 수정 ──────────────────────────────────────────
  · Civilian이 플레이어에게 밀리는 현상 수정 (Rigidbody2D → Kinematic).
  · VisionCone 매 프레임 GC 할당 제거 (배열 캐시).
  · TMP enableWordWrapping deprecated 경고 수정 (textWrappingMode로 교체).
  · NpcKindData 불필요 kind 필드 제거.

  2026-06-11  (세션 3) — 타원형 그림자 시스템 + 미니맵
================================================================

  ── 그림자 시스템 전면 교체 (EllipseShadow) ──────────────────

  · [구조 변경] ShadowProjector: 스프라이트 복사 방식 → 타원형 그라데이션으로 교체.
    이전: 원본 오브젝트 스프라이트를 그대로 복사한 _Shadow GO (판정 범위가 오브젝트
          모양 그대로라 직관적이지 않음).
    변경: 중심→가장자리 그라데이션 원형 스프라이트 사용. localScale(shadowFlat, length)
          으로 타원형 스케일 → 시각적으로 자연스러운 땅 그림자 표현.

  · [신규] EllipseShadow.cs 컴포넌트 제작.
    - 수학적 타원 방정식으로 플레이어 은신 판정 (Physics2D 콜라이더 불필요).
    - ContainsPoint(): (lx/rx)² + (ly/ry)² ≤ 1, transform.rotation 반영.
    - createVisual=true  (독립 배치): radiusX/radiusY로 크기 지정, 비주얼 자동 생성.
    - createVisual=false (ShadowProjector 연동): lossyScale/2를 반경으로 자동 사용.
      → ShadowProjector가 매 프레임 _Shadow GO 크기를 조정하면 판정도 자동 동기화.
    - OnEnable/OnDisable: EllipseShadow.All 정적 리스트에 등록/해제.

  · [수정] PlayerExposureTracker.cs: Physics2D 기반 감지 → EllipseShadow.All 순회로 교체.
    - shadowLayer LayerMask / _hitBuffer / _zoneCounts / ContactFilter2D 필드 제거.
    - Tick(): 5점 샘플 각각을 EllipseShadow.All 순회 → 3점 이상 포함 시 Shadow 판정.

  · [수정] ShadowProjector.cs: ShadowZone + Rigidbody2D + Collider2D 생성 코드 제거.
    createHidingZone=true 시 EllipseShadow(createVisual=false) 자동 부착.
    그라데이션 스프라이트 PPU를 원본 스프라이트 월드 크기에 맞춰 역산
    (gradPPU = 64 / srcMax) → 시각적 크기가 기존과 동일하게 유지.

  ── 그림자 시각 품질 개선 ────────────────────────────────────

  · 그라데이션 공식 변경: (1-t)² → 1-t²
    이전: 중심에서 빠르게 감쇠 → 전체적으로 흐릿하게 보임.
    변경: 중심 면적을 넓게 유지하고 가장자리만 부드럽게 페이드
          → 더 진하고 명확한 그림자.

  · shadowAlpha 기본값 상향: 0.45 → 0.75
    씬 내 기존 ShadowProjector 122개 Inspector 값 일괄 업데이트.

  ── 미니맵 시스템 신규 구현 (MinimapController.cs) ───────────

  · Tilemap 범위 자동 감지 → 전체 맵을 커버하는 Orthographic 카메라 생성.
    카메라 위치: 맵 중심 (8.5, -70.5, -100), orthographicSize=48.5, aspect=1.33.

  · RenderTexture (512×386) → MiniMapPanel 하위 RawImage(_MapView)에 출력.
    패널 가로세로 비율에 맞춰 RT 크기 자동 계산.

  · 플레이어/NPC 위치 아이콘 실시간 오버레이 (_IconRoot).
    아이콘 좌표: Camera.WorldToViewportPoint() → 패널 anchoredPosition 변환.
      - 플레이어  : 흰색 원형 아이콘
      - Guard     : 붉은색 / Alert·Chase 상태 시 밝은 빨강으로 변화
      - Civilian  : 파란색
    RectMask2D로 패널 경계 밖 아이콘 자동 클리핑.

  · NPCController에 public NpcKind 프로퍼티 추가 (npcType 외부 노출용).
